Что изменено в проекте

1. Ручное обновление переведено в фоновый режим.
   Кнопка больше не должна подвешивать страницу, потому что тяжёлый парсер не выполняется внутри HTTP-запроса.

2. Добавлен статус обновления на сайте.
   Есть отдельный API /api/update-status и визуальный блок со статусом последнего запуска.

3. Добавлено ежедневное автообновление.
   Планировщик запускает обновление каждый день в 10:00 по Asia/Vladivostok.

4. Убран блок выбора дат / календарь.
   Сайт показывает только последнее успешное обновление.

5. Старая история очищается сразу.
   После нового сохранения остаётся только текущий файл истории и текущий snapshot.

6. Усилены проверки данных.
   Перед сохранением проверяются набор валют, дубли, обязательные курсы, базовые даты источников и целостность payload.

7. Добавлены сетевые retry для requests.
   Это делает загрузку устойчивее к кратковременным сбоям источников.

Запуск

1. Установить зависимости:
   pip install -r requirements.txt

2. Для Playwright при первом запуске может понадобиться:
   playwright install chromium

3. Локальный запуск:
   python app.py

4. Продакшн-запуск:
   ./run.sh

Важно

Автообновление работает внутри процесса приложения. Если хостинг «усыпляет» приложение или процесс не запущен в 10:00 по Владивостоку, плановый запуск в это время не произойдёт.
